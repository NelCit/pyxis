# ci-golden-file-check

CI runs three golden-file gates that fail loudly. They aren't "human errors" to be silenced — they signal contract changes that demand a coordinated response.

## Gate 1 — `_tools/golden_exports.txt` (§22.4)

**What it does.** `_tools/check_exports.py` runs `dumpbin /exports pyxis_renderer.dll`, filters to public symbols, and diffs against `_tools/golden_exports.txt`.

**A diff fails the build unless** (a) the PR also updates the golden file and (b) `version.txt` minor or major is bumped.

When CI flags this:

1. Run `_tools/check_exports.py` locally; capture the diff.
2. Classify: are symbols **added** (MINOR-eligible), **removed** (MAJOR), or **renamed** (MAJOR — needs the §22.3 two-MINOR deprecation window)?
3. Confirm the §22.3 deprecation window for any removal:
   - **N** — introduce new symbol, mark old one `[[deprecated("use Foo instead; removed in 2.0")]]`. Both must work and be tested.
   - **N+1** — leave deprecated, log usage at info level if `PYXIS_DEBUG_TOOLS`.
   - **MAJOR bump** — remove.
4. If the diff is non-additive, the PR title must contain `[abi-break]` (§22.3).
5. Update `_tools/golden_exports.txt` to match the new dumpbin output.
6. Update `version.txt` per §22.1:
   - **MAJOR**: signature removed/renamed, `sizeof` changed without reserved consumption, enum renumbered, handle layout changed
   - **MINOR**: new method, new POD field consuming a `_reservedN` slot, new `ErrorKind` at end of enum, new optional `RenderTargets` AOV slot
   - **PATCH**: bugfix only, no API changes, no behavioural changes that move regression RMSE outside per-test tolerance
7. Update the symbol-version map (`pyxis_renderer.def`).

If unsure whether a change is MINOR or MAJOR, run `/api-surface-check` first.

## Gate 2 — `version.txt`

**What it does.** Generated header `Public/Pyxis/Renderer/Version.h` derives `PYXIS_VERSION_MAJOR/MINOR/PATCH/STRING/ENCODED` from `version.txt` via `_cmake/Version.cmake`.

**Runtime check (§22.2).** `Application::Init` calls `GetVersionEncoded()` and asserts `(macro_at_compile >> 16) == (runtime >> 16)`. Major+minor must match between consumer compile and loaded DLL. PATCH mismatch is logged at info only.

When CI flags a version mismatch:

1. The git SHA is **not** a preprocessor macro — it lives in a `.cpp` generated by `_cmake/Version.cmake` and is exposed via `pyxis::GetVersionGitSha()`. Don't add it to a header.
2. Confirm `version.txt` is bumped before the export-diff PR merges; bumping after the fact creates a release-window with broken consumers.

## Gate 3 — `NOTICE.generated` / SBOM (§46.3, §48.3)

**What it does.** When a third-party dependency is added or upgraded, `vcpkg.json` baseline and `_cmake/Thirdparty.cmake` SHA must be updated *together*. License audit (`_tools/license_audit.py`) regenerates `NOTICE.generated` from the resolved dependency tree.

When CI flags a NOTICE diff:

1. List the new/changed deps from the `vcpkg.json` diff.
2. Confirm each license is in the allowlist (Apache-2.0, MIT, BSD-2/3-Clause, ZLib, Boost — Pyxis ships under Apache-2.0; copyleft introduces compliance burden).
3. Run `_tools/license_audit.py` locally; commit the regenerated `NOTICE.generated`.
4. If a dependency went to a new MAJOR (e.g. OpenUSD v25 → v26), an RFC is required (§44.1). Stop and prompt the user to point at the RFC before merging.
5. SBOM is deferred to v1.1 (§46.3) — the scaffolding is in place but not enforced yet.

## Gate 4 — symbol-version map (`pyxis_renderer.def`)

Generated from public headers, diffed in CI (§22.3). A non-additive diff fails the build unless the PR title contains `[abi-break]` AND `version.txt` is bumped to a new MAJOR.

## Output

```
## Golden-file diffs detected

### _tools/golden_exports.txt
- [+] pyxis::PyxisRenderer::PickAt(int, int)  → additive, MINOR-eligible
- [-] pyxis::Profiler::OldScope(const char*)  → removal; deprecation window not started — block

### Required actions
1. Restore Profiler::OldScope and mark [[deprecated]]; ship for one MINOR before removal (§22.3)
2. After both above are clean: bump version.txt to 1.1.0 and update _tools/golden_exports.txt
3. Update pyxis_renderer.def

### NOTICE.generated
- New dep: zstd 1.5.6 (BSD-2-Clause) — allowlist OK; regenerate NOTICE.generated
- vcpkg.json baseline updated; _cmake/Thirdparty.cmake SHA matches: OK
```

Don't auto-edit `version.txt` or the golden files without confirming the classification with the user — these are PR-blocking and a wrong bump is a release issue.
